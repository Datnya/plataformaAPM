# Estructura de Base de Datos Relacional

## 1. Tabla: `Users` (Usuarios)
- `id` (UUID)
- `email` (String, único)
- `role` (Enum: ADMIN, CONSULTOR, CLIENTE)
- `name` (String)
- `company_id` (Relación con tabla Clients, null si es Admin/Consultor)

## 2. Tabla: `Clients` (Empresas Cliente)
- `id` (UUID)
- `company_name` (String)
- `status` (String: Activo, Inactivo)

## 3. Tabla: `Projects` (Proyectos)
- `id` (UUID)
- `client_id` (Relación con Clients)
- `consultant_id` (Relación con Users)
- `start_date` (Date)
- `end_date` (Date)

## 4. Tabla: `TimeLogs` (Registro de Jornada y Evidencias)
- `id` (UUID)
- `project_id` (Relación con Projects)
- `consultant_id` (Relación con Users)
- `date` (Date)
- `check_in_time` (Timestamp)
- `check_out_time` (Timestamp)
- `modality` (Enum: PRESENCIAL, REMOTO)
- `areas_visited` (Array de Strings)
- `people_met` (JSON: [{nombre, apellido, cargo}])
- `evidence_urls` (Array de Strings - URLs de archivos)

## 5. Tabla: `Goals` (Objetivos)
- `id` (UUID)
- `project_id` (Relación con Projects)
- `description` (String)
- `type` (Enum: SEMANAL, MENSUAL)
- `is_completed` (Boolean)

## 6. Tabla: `Prospects` (Control de Prospectos - CRM)
- `id` (UUID)
- `company_name` (String)
- `contact_name` (String)
- `first_contact_date` (Date)
- `last_interaction_date` (Date)
- `status` (Enum: NUEVO, CONTACTADO, NEGOCIACION, CERRADO)
